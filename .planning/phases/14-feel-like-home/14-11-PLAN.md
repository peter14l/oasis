# 14-11: Digital Will

**Feature:** Digital Will - Legacy Planning  
**Phase:** 14 - Feel Like Home  
**Status:** Planning  

---

## TASK

Designate what happens to your digital presence after a period of inactivity. Legacy contact gets access, messages auto-delete, data transferred. For responsible planning.

---

## PLAN

### Step 1: Configuration
1. Set legacy contact (must be OASIS user)
2. Set inactivity period (30, 60, 90, 365 days)
3. Choose what happens:
   - All data transferred
   - Messages deleted
   - Account memorialized

### Database
```sql
CREATE TABLE digital_wills (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  legacy_contact_id UUID REFERENCES auth.users(id),
  inactivity_days INTEGER DEFAULT 90,
  data_action VARCHAR(20) DEFAULT 'transfer',  -- transfer, delete, memorialize
  message_action VARCHAR(20) DEFAULT 'delete',
  confirmed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Step 2: Check Logic
1. Daily job checks last_active
2. If inactive > threshold → trigger will
3. Notify user BEFORE executing (grace period)

### Step 3: Execution
- Transfer: grant read access to legacy contact
- Delete: permanent removal
- Memorialize: account "remembered" state

### Step 4: UI
1. Settings → Digital Will section
2. Setup wizard
3. Confirmation required
4. Annual reminder to review
