# 14-01: Mood Orbit - Personal Vibe Widget

**Feature:** Mood Orbit - Personal Vibe Widget  
**Phase:** 14 - Feel Like Home  
**Status:** Planning  

---

## TASK

Create a personal "vibe" widget that shows the user's current mood/state on their profile and feed. Others see it before engaging, creating a warm, expressive "home" feeling.

---

## CONTEXT

### Existing to Build On
- User profile: `lib/features/profile/`
- Providers: `lib/features/feed/presentation/providers/`
- Supabase client already in use

### Database Changes
```sql
-- New table for user moods
CREATE TABLE user_moods (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  mood VARCHAR(50) NOT NULL,
  custom_label VARCHAR(100),  -- for custom moods
  visibility VARCHAR(20) DEFAULT 'friends',  -- everyone, friends, none
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Predefined moods
INSERT INTO user_moods (user_id, mood) VALUES 
  (DEFAULT, 'calm'), (DEFAULT, 'energetic'), (DEFAULT, 'need coffee'),
  (DEFAULT, 'creative'), (DEFAULT, 'introspective'), (DEFAULT, 'focused'),
  (DEFAULT, 'restless'), (DEFAULT, 'content'), (DEFAULT, 'inspired');
```

---

## PLAN

### Step 1: Database & Schema
1. Create `user_moods` table with RLS policies
2. Add mood column to `profiles` table (current_mood, mood_updated_at)
3. Set up Row Level Security:
   - Users can read friend's moods
   - Anyone can read if visibility = 'everyone'
   - Block if visibility = 'none'

### Step 2: Backend API
1. Create Supabase function `set_mood(mood, visibility)`
2. Create function `get_friends_moods(user_ids)`
3. Create function `get_available_moods()`

### Step 3: Provider
1. Create `MoodProvider` in `lib/features/feed/presentation/providers/`
   - `currentMood` state
   - `setMood(mood, visibility)` method
   - `availableMoods` list
   - `customMoods` list
2. Hook into existing providers

### Step 4: UI - Mood Picker
1. Create `MoodPicker` widget in `lib/widgets/mood/`:
   - Grid of mood icons/emojis
   - Custom mood option (text input)
   - Visibility toggle
2. Place in:
   - Profile header (tap mood to change)
   - Quick action from feed

### Step 5: UI - Feed Integration
1. Show mood on user's post card (small indicator)
2. Show mood when composing DM: "Sending to [name] who's feeling [mood]"
3. Show mood on chat header

### Step 6: Persistence
1. Store last mood selection locally (SharedPreferences)
2. Auto-suggest last mood on app open
